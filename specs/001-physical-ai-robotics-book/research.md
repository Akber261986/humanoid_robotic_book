## Research Findings for Educational Book on Physical AI & Humanoid Robotics

This document consolidates research on technical aspects identified as 'NEEDS CLARIFICATION' during the planning phase for the educational book on Physical AI and Humanoid Robotics.

### 1. Recommended Python Version for Robotics Integration

*   **Decision**: Python 3.11
*   **Rationale**: While ROS 2 supports various Python 3.x versions (e.g., Python 3.10 with Ubuntu 22.04 for Humble, Python 3.12 with Ubuntu 24.04 for Jazzy), NVIDIA Isaac Sim explicitly requires **Python 3.11**. To ensure seamless integration and leverage Isaac Sim's internal ROS capabilities, Python 3.11 is the most straightforward and fully supported choice. External ROS nodes can use other Python versions as DDS handles data transport, but for internal Isaac Sim ROS, 3.11 is key.
*   **Alternatives considered**: Python 3.10, Python 3.12 (less direct support for Isaac Sim in 2025-12).

### 2. Common Python Libraries and APIs for VLA Integration with Robotics

*   **Decision**: Focus on NVIDIA's OpenVLA, Isaac Sim, Isaac Lab, and specialized Isaac ROS 2 packages for core VLA-robotics integration. Complement with general-purpose libraries like OpenCV and PyTorch for broader AI/CV tasks.
*   **Rationale**: NVIDIA provides a tightly integrated ecosystem for AI-driven robotics. OpenVLA is specifically designed for embodied robotics and leverages LLMs/VLMs. Isaac Sim and Isaac Lab offer robust simulation and learning environments with strong ROS 2 support. Specialized Isaac ROS 2 packages provide GPU-accelerated functionalities for perception, navigation, and DNN inference. General libraries like OpenCV and PyTorch are essential for broader computer vision and deep learning needs within this ecosystem.
*   **Alternatives considered**: Other general-purpose LLM/VLM APIs (e.g., direct OpenAI, Hugging Face `transformers`) can be integrated, but NVIDIA's offerings provide more direct robotics-specific integrations.

### 3. Common Python Scientific Libraries for Robotics Applications

*   **Decision**: Utilize NumPy and SciPy for foundational numerical operations and scientific computing. Integrate OpenCV for computer vision, PyTorch/TensorFlow for deep learning, and potentially Scikit-learn for traditional ML. Matplotlib/Seaborn and Pandas for data visualization and analysis.
*   **Rationale**: These libraries form the backbone of scientific computing and AI in Python. NumPy and SciPy provide efficient numerical handling. OpenCV is standard for computer vision in robotics. PyTorch/TensorFlow are essential for advanced AI models, especially for perception and control. Scikit-learn offers a wide range of traditional ML algorithms for various robotic tasks. Matplotlib/Seaborn and Pandas are crucial for data analysis and visualization, especially for post-simulation analysis.
*   **Alternatives considered**: Specialized domain-specific libraries exist but these general-purpose ones offer broad applicability and strong community support.

### 4. Recommended Python Unit Testing Frameworks for Robotics Development

*   **Decision**: `pytest` as the primary unit testing framework due to its flexibility and robust fixture system. `unittest` can be used for simpler, self-contained components.
*   **Rationale**: `pytest` offers a simpler syntax, powerful fixtures for managing complex test environments (e.g., mocking ROS 2 nodes, sensor data), and a rich plugin ecosystem. This is highly beneficial for the intricate setups in robotics. `unittest` is a viable alternative for basic component testing.
*   **Key Practices**: Modular design, mocking and simulation independence (using `unittest.mock` for ROS 2 interfaces, sensor data, actuator commands), comprehensive test data including edge cases, and focusing on logic independent of the environment.
